# Contributing

First of all, thank you for considering contributing to the Opportunity Tracker! 

This project started from a simple frustration: amazing academic and professional opportunities are spread across dozens of websites, and many students only discover them after the application deadline has already passed.

The goal is to build something much bigger than a collection of calendars.

I hope this becomes a collaborative knowledge base where students help other students discover opportunities, prepare better applications, and share everything they learned along the way.

Even a small contribution can save someone from missing an incredible opportunity.

---

# How can I contribute?

There are many ways to help, and **you don't need to write code** to make a meaningful contribution.

## 📅 Add new opportunities

The easiest way to contribute is by adding new opportunities.

Examples include:

* internships;
* undergraduate programs;
* master's and PhD opportunities;
* summer schools;
* conferences;
* workshops;
* scholarships;
* fellowships;
* competitions;
* exchange programs;
* research internships;
* industry programs.

If you're unsure whether something fits the project, open an Issue and ask!

---

## ✏️ Correct existing information

Information changes all the time.

You can help by correcting:

* application deadlines;
* broken links;
* eligibility requirements;
* locations;
* dates;
* names;
* descriptions.

Keeping information up to date is just as valuable as adding new opportunities.

---

## 📚 Write tutorials and guides

One of the long-term goals of this repository is to become a place where students can also learn **how to prepare** for these opportunities.

Examples of guides include:

* How to prepare for a summer school
* How to write a Statement of Purpose
* How to request recommendation letters
* How to prepare your CV
* How to contact professors
* Application timelines
* Visa tips
* Funding advice
* Interview preparation
* Common mistakes during applications

If you know something that could make someone else's application easier, consider writing about it.

---

# Share your experience

This is probably the most valuable contribution you can make.

If you have participated in any opportunity—whether it was a conference, internship, exchange program, research experience, scholarship, summer school, or anything similar—we would love to hear about it.

Consider creating a new Markdown file inside `tutorials-and-info/` sharing your experience.

Some ideas of what you could include:

* how you found the opportunity;
* application timeline;
* selection process;
* interviews;
* documents requested;
* preparation strategy;
* what surprised you;
* things you wish you had known beforehand;
* travel tips;
* funding information;
* accommodation;
* lessons learned;
* advice for future applicants.

You don't need to write a perfect guide.

Even a few paragraphs can help someone tremendously.

---

# Share useful resources

Templates and examples are incredibly helpful for students applying for opportunities for the first time.

Useful resources include:

* CV templates;
* academic CV examples;
* résumé examples;
* Statement of Purpose examples;
* Motivation Letter examples;
* Personal Statement examples;
* Research Statement examples;
* recommendation letter request emails;
* application checklists;
* document checklists;
* preparation timelines.

If you share your own material, please remove personal or sensitive information whenever appropriate.

Also make sure you have the right to share any document before uploading it.

If you enjoy programming, there are plenty of technical contributions as well.

You can help by:

* fixing bugs;
* improving the documentation;
* improving the GitHub Pages website;
* adding new parsers;
* improving CSV validation;
* improving the CLI;
* improving calendar generation;
* adding automated tests;
* improving GitHub Actions;

# Specialized forks are welcome

The official repository aims to cover opportunities that are broadly useful.

However, you may want something more specific.

Feel free to fork the project and build calendars for:

* your university;
* your research group;
* your country;
* a specific field of study;
* a student organization;
* your community.

If your improvements could benefit everyone, consider opening a Pull Request afterward.

---

# Opening an Issue

If you notice:

* incorrect information;
* missing opportunities;
* broken links;
* confusing documentation;
* ideas for improvements;
* bugs;

please open an Issue.

You don't need to have the solution.

Reporting a problem is already a valuable contribution.

---

# Opening a Pull Request

For code or content contributions:

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Commit your work.
5. Open a Pull Request.

Please keep Pull Requests focused on a single topic whenever possible.

Small PRs are much easier to review than very large ones.

---

# Community values

This project is built around one simple idea:

> Knowledge grows when it is shared.

Whether you're fixing a typo, adding a deadline, sharing your application story, or contributing new features, you're helping future students discover opportunities they might otherwise never have found.

Thank you for helping make this project better. 🚀
